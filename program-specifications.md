## 📋 Компоненти системи

| Модуль               | Відповідальність                                                                 |
|----------------------|----------------------------------------------------------------------------------|
| **UI**               | Користувацький інтерфейс, взаємодія з користувачем, відображення результатів     |
| **GraphController**  | Головний контролер, маршрутизація запитів, координація роботи всіх компонентів   |
| **Graph**            | Модель графа, зберігання вершин та ребер, операції над структурою даних          |
| **AlgorithmEngine**  | Фабрика та виконавець алгоритмів, загальний інтерфейс для пошуку шляхів         |
| **Dijkstra**         | Реалізація алгоритму Дейкстри для пошуку найкоротшого шляху                      |
| **BellmanFord**      | Реалізація алгоритму Беллмана-Форда для графів з від'ємними вагами               |
| **A***               | Реалізація евристичного алгоритму A* з орієнтацією на мету                       |
| **PathResult**       | Модель результату пошуку шляху, зберігання маршруту, ваги, статистики            |
| **ExportService**    | Сервіс експорту результатів у різні формати (зображення, текст, JSON)            |
| **GraphVisualizer**  | Візуалізація графа та знайденого шляху, графічний вивід                          |
| **BenchmarkRunner**  | Заміри продуктивності алгоритмів, порівняння часу виконання, звітність           |

---

## 🔄 Потік виконання запиту

1.  Користувач вводить параметри через **UI**
2.  Запит передається до **GraphController**
3.  Controller завантажує/будує структуру **Graph**
4.  Controller передає граф та параметри в **AlgorithmEngine**
5.  AlgorithmEngine викликає вибраний алгоритм (Dijkstra / BellmanFord / A*)
6.  Алгоритм обчислює шлях та повертає об'єкт **PathResult**
7.  PathResult передається до:
    -   **GraphVisualizer** для візуалізації
    -   **BenchmarkRunner** для замірів продуктивності
    -   **ExportService** для збереження результату
8.  Кінцевий результат відображається користувачеві через UI

## 🛡️ Принципи архітектури

✅ Один відіальність (Single Responsibility)
✅ Розділення інтерфейсів
✅ Залежності тільки на вищих рівнях
✅ Зручність розширення новими алгоритмами
✅ Незалежність модулів один від одного
